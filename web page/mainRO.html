O functie reprezinta un ansamblu de linii de cod, care implementeaza intr-o forma riguroasa un algoritm. Algoritmul din functia respectiva poate fi refolosit de cate ori este nevoie. Functiile se mai numesc subprograme si rezolva o subproblema a aplicatiei noastre. Intr-un program C++, o functie se defineste si apoi “se apeleaza” din functia principala, dintr-o alta functie sau chiar din aceeasi functie.

Forma generala a definirii unei functii este: 

tip de date [returnat] nume.functie (lista parametrilor formali)
{
      instructiuni;
      [return expresie]
}

Tipul de date poate fi oricare din cele studiate si reprezinta, cu o singura exceptie, tipul de date al expresiei din return. Atunci cand tipul de date al functiei este void, functia nu va returna ceva si atunci intructiunea return expresie lipseste.

Lista parametrilor formali este compusa din totalitatea variabilelor pentru care se va executa functia. Parametrii formali se scriu impreuna cu tipul lor de date, iar daca sunt mai multi se vor separa prin “;”.

Numele functiei nu contine spatii si nu reprezinta denumiri de instructiuni sau cuvinte cheie din C++. Atunci cand numele functiei este compus din 2 sau mai multe functii, se va urma urmatorul tipar: primul cuvant se scriu cu minuscule, iar urmatoarele se scriu legate cu initcap. ex: nume , sumaCifreNr, suma_cifre

Folosirea unei functii intr-un program se realizeaza prin intermediul unui apel. Apelul reprezinta executia pas cu pas pentru anumite valori.

Sintaxa unui apel: 

nume.functie(lista parametrilor efectivi)[;]

Lista parametrilor efectivi reprezinta variabilele sau valorile pentru care se executa functia. Acesti parametri se scriu fara tipul lor de date si fara dimensiune daca este cazul. Separarea lor se face tot printr-un singur spatiu.

Executia unei functii se opreste dupa rularea instructiunilor return expresie, indiferent de cate linii de cod urmeaza.

In momentul apelului se verifica urmatoarele concordante:

  • se verifica daca numele functiei din apel coincide cu numele unei functiei definte inainte de main;
  • tipul de date al parametrilor efectivi coincide cu numarul parametrilor formali;
  • tipul de date al parametrilor efectivi trebuie sa fie acelasi cu tipul de date al parametrilor formali

    • Parametrii efectivi transmit parametrilor formali valorile in ordine si, in acest fel, parametri efectivi isi pastreaza valorile lor.
